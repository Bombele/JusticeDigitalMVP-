# 06 - Arquitectura IA + informática / Architecture IA + informatique

## ES — Arquitectura IA + informática

JusticeDigitalMVP combina Inteligencia Artificial y ciencias informáticas para construir una plataforma ciudadana de justicia digital.  
Los componentes principales son:

- **FastAPI**: framework ligero y robusto para crear endpoints REST.  
- **Pydantic**: validación de datos y modelos.  
- **spaCy**: procesamiento de lenguaje natural (NLP) para anonimización automática de entidades sensibles.  
- **SQLAlchemy**: ORM para definir modelos y gestionar persistencia.  
- **SQLite**: base de datos inicial para prototipos; migración prevista a **PostgreSQL** en producción.  
- **Frontend**: formulario HTML simple, con posibilidad de evolucionar hacia React o Streamlit.  
- **Seguridad**: uso de `.env` para secretos, autenticación básica (JWT), cifrado en reposo y control de accesos.  
- **Infraestructura**: despliegue en Railway y Hugging Face Spaces, con documentación en GitHub.  

La arquitectura está diseñada para ser **ética, abierta y escalable**, permitiendo que cada denuncia se convierta en memoria activa y resiliente.

## FR — Architecture IA + informatique

JusticeDigitalMVP combine Intelligence Artificielle et sciences informatiques pour construire une plateforme citoyenne de justice numérique.  
Les composants principaux sont :

- **FastAPI** : framework léger et robuste pour créer des endpoints REST.  
- **Pydantic** : validation des données et modèles.  
- **spaCy** : traitement du langage naturel (NLP) pour l’anonymisation automatique des entités sensibles.  
- **SQLAlchemy** : ORM pour définir les modèles et gérer la persistance.  
- **SQLite** : base de données initiale pour les prototypes ; migration prévue vers **PostgreSQL** en production.  
- **Frontend** : formulaire HTML simple, avec possibilité d’évolution vers React ou Streamlit.  
- **Sécurité** : utilisation de `.env` pour les secrets, authentification basique (JWT), chiffrement au repos et gestion des accès.  
- **Infrastructure** : déploiement sur Railway et Hugging Face Spaces, avec documentation sur GitHub.  

L’architecture est conçue pour être **éthique, ouverte et évolutive**, permettant que chaque dénonciation devienne une mémoire active et résiliente.
